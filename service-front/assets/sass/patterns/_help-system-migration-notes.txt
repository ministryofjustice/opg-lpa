Migration notes: _help-system.scss — removal of legacy govuk_frontend_toolkit identifiers
========================================================================================
Date: 2026-06-01

Overview
--------
All direct references to govuk_frontend_toolkit variables and mixins have been removed
from _help-system.scss and replaced with standard CSS equivalents. This is part of the
wider GOV.UK Frontend v5 migration (Phase 2 — bespoke SCSS catch-up) described in
docs/govuk-frontend-migration-plan.md.

No visual behaviour has been changed; all replacement values are exact equivalents of
the toolkit originals.


1. Variables replaced
---------------------
The following toolkit variables were substituted with their hardcoded CSS values:

  $gutter-half   →  15px      (toolkit value: $gutter / 2 = 15px)
  $gutter        →  30px      (toolkit value: 30px)
  $border-colour →  #bfc1c3   (toolkit value: $border-colour)
  $link-colour   →  #005ea5   (toolkit value: $link-colour)
  $panel-colour  →  #dee0e2   (toolkit value: $panel-colour)

Occurrences changed:
  .help-header          padding, border-bottom
  .help-body            .column-left — padding, border-right
  .help-body            .column-right — padding
  .help-navigation      padding
  .help-topics          margin-top
  .help-topics .active  border-left, background-color
  .help-sections        padding (popup variant)
  .help-footer          padding, border-top, hr margin


2. Mixins replaced
------------------
2a. @include box-sizing(border-box)
    Replaced with the standard CSS property directly:
      box-sizing: border-box
    Occurrences: .help-body .column-left, .help-body .column-right

2b. @include media(...) — toolkit responsive media mixin
    Replaced with native @media queries. Exact breakpoint values are preserved.

    @include media(mobile)
      →  @media (max-width: 640px)
         (toolkit "mobile" = max-width 640px)
    Occurrences: .help-header-title, .help-body .column-left,
                 .help-body .column-right, .help-navigation-title,
                 .help-topics li (popup), .help-sections (popup),
                 .help-topics a

    @include media($min-width: 640px, $max-width: 1000px)
      →  @media (min-width: 640px) and (max-width: 1000px)
    Occurrences: .help-body .column-left, .help-body .column-right

    @include media($min-width: 641px)
      →  @media (min-width: 641px)
    Occurrences: .help-topics .active (popup), .help-sections article,
                 .link-back (popup)

    @include media($max-width: 640px)
      →  @media (max-width: 640px)
    Occurrences: .help-topics a

    @include media($min-width: 640px)
      →  @media (min-width: 640px)
    Occurrences: .help-sections article:first-child h2


3. What was NOT changed
-----------------------
The following @extend references were left unchanged because the placeholders they
reference are defined in the project's own extension files
(assets/sass/extensions/elements/_elements-typography.scss and _lists.scss),
not directly from the toolkit:

  @extend %heading-large
  @extend %heading-xlarge
  @extend %heading-medium
  @extend %heading-small
  @extend %list
  @extend %list-bullet
  @extend %list-number
  @extend %list-alpha

Migrating these placeholders (and their internal use of toolkit mixins such as
bold-36(), bold-48(), bold-24(), bold-19()) is a separate concern tracked under
the same Phase 2 work for assets/sass/extensions/elements/*.


4. Next steps
-------------
To fully remove the legacy toolkit from the project, the remaining files still
referencing toolkit identifiers must also be migrated (see migration plan section 1.1),
and then Phase 3 cleanup can run:
  - Remove @import "govuk-elements" from application.scss
  - Remove $govuk-compatibility-govukfrontendtoolkit and $govuk-compatibility-govukelements
    from settings/_variables.scss
  - npm uninstall govuk_frontend_toolkit govuk_template_mustache govuk-elements-sass
